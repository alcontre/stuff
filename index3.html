Binary Data Format Description

File Header

  • Signature: magic bytes indicating format identity.
  • Version: 16-bit integer identifying schema revision.

Chunk Table

  • Chunk Entry: repeated entries describing each payload chunk.
    • Chunk Type: enumerated value describing purpose.
    • Chunk Offset: 32-bit offset from file start.
    • Chunk Size: 32-bit length in bytes.

Payload

  • Metadata Block: optional key/value pairs with UTF-8 strings.
  • Data Block: one or more records of the primary content.
    • Record Header: flags and length for each record.
    • Record Body: raw bytes interpreted per chunk type.
