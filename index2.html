Binary Format Structure

▼ Block: File Header
Bit Layout Description
[File Header Block: Variable]
Top-level metadata wrapper containing the magic number, versioning info, and ancillary metadata records.
▼ Block: Magic Number
Bit Layout Description
[Magic Number Block: 4B]
Fixed identifier that guards the file and enables format sniffing before deeper parsing.
▼ Block: Magic Number
Bit Layout Description
[Magic Number Block: 4B]
Fixed identifier that guards the file and enables format sniffing before deeper parsing.
▼ Block: Signature Bytes
Bit Layout Description
[Signature Bytes: 4 ASCII]
Literal byte pattern that must match the expected "BINF" sequence.
▼ Block: ASCII Representation
Bit Layout Description
"BINF"
ASCII encoding of magic bytes (0x42 0x49 0x4E 0x46)
▼ Block: Version
Bit Layout Description
[Version Block: 2B]
Encodes major and minor values that determine the parsing rules for the rest of the structure.
▼ Block: Major Version
Bit Layout Description
[Major Version Entry: 1B]
Holds the compatibility-breaking revision component.
▼ Block: Version Number
Bit Layout Description
[Version: 1B]
Major version number (e.g., 0x01 for version 1)
▼ Block: Minor Version
Bit Layout Description
[Revision: 1B]
Minor version/revision number
▼ Block: Metadata
Bit Layout Description
[File Size: 4 bytes]
Total file size in bytes
[Checksum: 4 bytes]
CRC32 checksum of file content
▼ Block: Data Section
Bit Layout Description
[Data Section: Variable]
Carries the repeating metadata and content blocks that make up the payload of the file.
▼ Block: Metadata Block
Bit Layout Description
[Metadata Block: Variable]
Optional block that precedes user content with descriptors, timestamps, and sizing info.
▼ Block: Block Header
Bit Layout Description
[Block Header: Variable]
Prefix for every block that conveys structural information like type, flags, and sizing.

▼ Block: Block Length
Bit Layout Description
[Length: 3B]
Block length in bytes (24-bit, max 16MB)
▼ Block: Timestamp
Bit Layout Description
[Timestamp: 8B]
Unix timestamp (64-bit seconds since epoch)
▼ Block: Content Block
Bit Layout Description
[Record Header: 2B]
Record type and flags (2 bytes)
[Record Data: Variable]
Variable-length record payload
[Compression: 1b]
Compression flag (0=uncompressed, 1=compressed)
▼ Block: Footer
Bit Layout Description
[Footer Block: Variable]
Terminal structures that provide lookup tables and final validation markers.
▼ Block: Index Table
Bit Layout Description
[Entry Count: 2B]
Number of index entries
[Entry Size: 1B]
Size of each entry in bytes
[Entries: 8B each]
Index entries (offset and length pairs)
▼ Block: End Marker
Bit Layout Description
[Terminator: 4B]
End-of-file marker (0xFFFFFFFF)
[Checksum: 4B]
Final verification checksum
